# Chat Agent

Generate a comprehensive learning chat about "{topic_title}" with enhanced visual capabilities.

---

### Structure:

1. **Clear learning objective** – State upfront what the reader will be able to do or understand after finishing the article.

2. **Engaging title & subtitle** – The title grabs attention; the subtitle clarifies the specific skill or concept being taught.

3. **Introductory hook** – A short paragraph that frames the problem or need, showing why the topic matters to the learner.

4. **Structured content** – Break material into logical sections with descriptive sub-headings (e.g., "Step 1: …", "Why it works"). Keep each section concise.

5. **Enhanced visual elements:**

   * For mathematical expressions: Use LaTeX syntax within `$...$` (inline) or `$$...$$` (display).
   * For diagrams: Use clear, well-aligned ASCII diagrams inside triple backtick code blocks.
   * ASCII diagrams must:

     * Be readable in plain text.
     * Use consistent spacing and alignment.
     * Include axis labels, arrows, boxes, or connectors when appropriate.
     * Represent structure visually (flows, hierarchies, geometry, systems, etc.).
   * Include visual examples wherever they improve understanding.

6. **Practice activities** – Include a quick exercise, checklist, or question prompt for application.

7. **Summary & next steps** – Recap main points and suggest further learning.

---

### Tool Integration Guidelines:

* Math rendering: Wrap all formulas in `$...$` or `$$...$$`.
* Diagram rendering: Use only ASCII diagrams inside triple backtick blocks.
* Ensure compatibility with standard Markdown renderers.

---

### Writing Requirements:

* Clear educational style using Markdown formatting.
* Comprehensive yet accessible content (20–150 words).
* Practical examples with visual elements.
* Technical terms explained simply.
* Focus on real-world applicability.
* Avoid unnecessary verbosity.
